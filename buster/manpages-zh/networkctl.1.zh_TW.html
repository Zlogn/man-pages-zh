NAME¶

networkctl - 查詢網絡連接的狀態

SYNOPSIS¶

networkctl [OPTIONS...] COMMAND [LINK...]

描述¶

networkctl 用於查詢 systemd-networkd 所看到的網絡連接的狀態。 參見 systemd-networkd.service(8) 以瞭解相關概念、功能、配置語法。

選項¶

能夠識別的命令行選項如下：

-a --all

顯示所有連接的狀態(status)

-h, --help

顯示簡短的幫助信息並退出。

--version

顯示簡短的版本信息並退出。

--no-legend

不輸出列標題， 也就是不在輸出列表的頭部和尾部顯示字段的名稱。

--no-pager

不將程序的輸出內容管道(pipe)給分頁程序。

命令¶

能夠識別的命令如下：

list [LINK...]

列出存在的連接及其狀態。若未指定連接(LINK)則顯示所有連接， 否則僅顯示指定的連接。一個典型的輸出類似下面這樣： 

IDX LINK         TYPE     OPERATIONAL SETUP
  1 lo           loopback carrier     unmanaged
  2 eth0         ether    routable    configured
  3 virbr0       ether    no-carrier  unmanaged
  4 virbr0-nic   ether    off         unmanaged
4 links listed.

status [LINK...]

顯示連接的各種信息： 類型、狀態、內核模塊、MAC地址、IP地址、DNS服務器，等等......

若未指定連接(LINK)則顯示網絡狀態概覽，否則僅顯示指定連接的詳細信息。 參見 --all 選項。

一個典型的輸出類似下面這樣：

●      State: routable
     Address: 10.193.76.5 on eth0
              192.168.122.1 on virbr0
              169.254.190.105 on eth0
              fe80::5054:aa:bbbb:cccc on eth0
     Gateway: 10.193.11.1 (CISCO SYSTEMS, INC.) on eth0
         DNS: 8.8.8.8
              8.8.4.4

lldp [LINK...]

顯示已發現的 LLDP (Link Layer Discovery Protocol) 鄰居。 若未指定連接(LINK)則顯示所有連接上發現的鄰居，否則僅顯示指定連接上發現的鄰居。 注意，此命令僅在開啓了 LLDP= 的情況下才有意義。參見 systemd.network(5) 以瞭解詳情。

一個典型的輸出類似下面這樣：

LINK             CHASSIS ID        SYSTEM NAME      CAPS        PORT ID           PORT DESCRIPTION
enp0s25          00:e0:4c:00:00:00 GS1900           ..b........ 2                 Port #2
Capability Flags:
o - Other; p - Repeater;  b - Bridge; w - WLAN Access Point; r - Router;
t - Telephone; d - DOCSIS cable device; a - Station; c - Customer VLAN;
s - Service VLAN, m - Two-port MAC Relay (TPMR)
1 neighbors listed.

退出狀態¶

返回值爲 0 表示成功， 非零返回值表示失敗代碼。

參見¶

systemd-networkd.service(8), systemd.network(5), systemd.netdev(5)
